# v1.1 后续阶段计划（联测并行）

更新时间：2026-02-26  
适用分支：`develop`

## 1. 目标

在等待前端联测反馈期间，明确后续开发顺序，补齐 `spec.md` 尚未完全覆盖能力，并准备发布收尾。

## 2. 执行原则

1. 严格 Git Flow：每个 Issue 独立分支、独立 PR。  
2. 先处理联测 P0，再补功能 P1/P2。  
3. 每个 Issue 必须附带自动化验证脚本。  
4. 发布前统一进行回滚演练。

## 3. 任务清单

### P0 联测缺陷收敛

#### I-043A 联测问题第一轮修复
- 输入：前端问题单（接口、字段、状态码、性能）
- 输出：按严重级别修复 PR
- DoD：
  - blocker/critical 全部清零
  - 相关回归脚本全部通过
  - 前端复测通过

---

### P1 spec 能力补齐（核心）

#### I-044 多时间粒度预测
- 目标：支持分/时/日/周窗口聚合与预测
- 输出：
  - 新增聚合参数（粒度、窗口）
  - 预测输出可按粒度切换
- DoD：
  - 至少 4 种粒度可用
  - 与现有 forecast 接口兼容

#### I-045 多路径 QoS 综合分析
- 目标：路径质量聚类与综合评分
- 输出：
  - 路径集输入
  - QoS 综合评分与排序
- DoD：
  - 支持多路径结果对比
  - 返回关键劣化指标解释

#### I-046 流量饱和与容量预警
- 目标：预测资源瓶颈时间并给出预警
- 输出：
  - 容量阈值模型
  - 预计饱和时间
- DoD：
  - 可输出未来瓶颈时间点
  - 支持动态阈值配置

#### I-047 安全事件与流量关联分析
- 目标：流量异常与安全事件库关联
- 输出：
  - 事件关联查询接口
  - 关联证据链
- DoD：
  - 支持按事件/流量双向查询
  - 结果包含置信度与证据

#### I-048 异常特征提取与解释
- 目标：输出异常特征信息供运维/前端展示
- 输出：
  - 特征摘要（时延突变/丢包跳变等）
  - 解释字段
- DoD：
  - 每条异常至少 1 条可解释证据

---

### P2 推演与复盘增强

#### I-049 演练报告生成
- 目标：自动生成推演复盘数据
- 输出：
  - 报告草稿 JSON
  - 关键事件时间线
- DoD：
  - 每次 simulation 可导出报告结构

#### I-050 播放控制增强
- 目标：补齐暂停/跳步/断点回溯能力
- 输出：
  - 控制接口（pause/resume/seek）
- DoD：
  - 前端可控 timeline 播放行为

## 4. 发布收尾（I-038）

### 发布前检查
1. 前后端联测清单全绿  
2. 压测与稳定性验证通过  
3. 回滚演练成功并留档

### 发布动作
1. 从 `develop` 切 `release/v1.1`  
2. 冻结变更，仅收敛缺陷  
3. 合并 `main` 并打 tag  
4. 回合 `develop`

## 5. 建议顺序

1. I-043A（联测缺陷）  
2. I-044 -> I-045 -> I-046 -> I-047 -> I-048  
3. I-049 -> I-050  
4. I-038 发布收尾
