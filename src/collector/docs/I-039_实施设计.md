# I-039 实施设计：AI安全事件联动分析

## 1. 目标
在现有故障分析链路中新增“安全事件联动”能力，将安全类告警与网络性能异常在同一时间窗内进行关联，输出结构化 `security_correlation` 给前端与AI解释模块使用。

## 2. 范围
1. 入站数据：`/api/v1/ingest/alarm` 增强支持 `category=security` 与 `security_tags[]`。
2. 后端分析：`/api/v1/bff/analysis/run` 新增 `security_correlation` 计算与返回。
3. 前端展示：故障分析报告区新增“安全联动”摘要块（等级、证据、疑似路径）。
4. 文档与测试：补充 API 文档、样例数据与回归脚本。

## 3. 数据结构设计
`analysis/run` 响应新增字段：

```json
"security_correlation": {
  "level": "none|low|medium|high",
  "evidence": [
    {
      "type": "time_window|entity_overlap|metric_anomaly|task_overlap",
      "score": 0.0,
      "detail": "..."
    }
  ],
  "suspected_path": ["NODE-A", "NODE-B", "NODE-C"],
  "window_sec": 300,
  "matched_security_events": 0
}
```

## 4. 计算逻辑（V1）
1. 时间窗关联：默认 `window_sec=300`，抓取最近安全事件。
2. 实体重叠：安全事件目标节点/链路与当前故障对象重叠得分。
3. 指标异常叠加：若同对象命中 loss/rtt/jitter/cpu/mem 异常，提升分值。
4. 任务重叠：受影响任务与安全事件涉及对象重叠，提升分值。
5. 等级映射：
   - score < 0.25 => none
   - [0.25, 0.5) => low
   - [0.5, 0.75) => medium
   - >= 0.75 => high

## 5. 代码改动计划
1. `src/collector/app/contract.py`
   - 扩展 alarm schema：`category`、`security_tags`、`security_event_id`（可选）
2. `src/collector/app/snapshot.py`
   - 存储安全事件的轻量索引（时间窗查询）
3. `src/collector/app/main.py`
   - 在 `analysis/run` 注入 `security_correlation`
   - 在 `analysis/explain` 的 `extra_context` 中透传安全联动结果
4. `src/frontend/src/App.jsx`
   - 分析报告区域增加安全联动展示块
5. `docs/docs/api/openapi_summary.md`
   - 更新相关字段说明与示例

## 6. 验收标准（DoD）
1. `analysis/run` 在有安全事件输入时返回 `security_correlation`。
2. 前端可显示等级与至少1条证据。
3. AI 报告可引用安全联动证据，不再只基于阈值信息。
4. 提供至少 10 条联动样例并通过脚本回归。

## 7. 风险与回滚
1. 风险：关联过度导致误报。
2. 缓解：阈值可配置、默认保守；输出 evidence 透明化。
3. 回滚：通过配置关闭 `security_correlation` 计算，仅返回 `level=none`。
